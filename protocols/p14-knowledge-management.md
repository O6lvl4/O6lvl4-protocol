# P14. 知識管理プロトコル

**目的**: 知的労働で得た知見を確実に蓄積し、複利的に再利用可能な状態に保つ。**「同じ問題を二度解かない」** を実現する。

## なぜ必要か

知的労働者にとって、「考えた成果」は揮発性が高い。書き留めなければ消え、書き留めても接続されなければ取り出せず、取り出せても定着しなければまた一から考えることになる。

知識管理は4つの層で成り立つ：

1. **キャプチャ** — 思考を消さずに捕まえる
2. **加工** — 1コンセプト1ノートに整理し、リンクで接続する
3. **定着** — SRS（間隔反復）で長期記憶に固定する
4. **再利用** — 検索・横断・合成で過去の自分を活用する

どこか1層が壊れると、そこから先の層は機能しない。

## 不変条件

- すべての知識インプット・アウトプットは **1つの inbox に集約**（分散禁止）
- **1ノート1コンセプト**（単一の Why に答える単位）
- inbox に投入したものは **24時間以内に加工**
- ノートは **必ず他ノートにリンクする**（孤立ノート禁止）
- **SRS のレビュー期日を遅延させない**

## パラメータ

| 層 | 操作 | 頻度 | 1回時間 |
|---|---|---|---|
| キャプチャ | inbox 投入 | 即時（思いついた瞬間） | < 30秒 |
| 加工 | inbox → vault 展開 | 毎日 | 10〜20分 |
| 定着 | SRS レビュー | 毎日 | 5〜15分 |
| 合成 | 接続発見・統合ノート作成 | 週1回 | 30〜60分 |
| 庭園整備 | 古いノート再訪・統廃合 | 月1回 | 60〜90分 |

## 実行手順

### キャプチャ（即時）

1. 学んだ・気づいた瞬間に inbox に1行で記録
2. 出典を付ける（URL、書籍、会話の文脈）
3. その場で加工しない。**捕獲を最優先**

### 加工（毎日、固定枠）

1. inbox を1件ずつ処理
2. 既存ノートの拡張で済むなら追記
3. 新規概念なら新ノート作成（1コンセプト原則）
4. 関連ノートに wikilink を張る
5. 「押さえどころ（カード化候補）」を3〜6個書き出す（SRS の種）
6. inbox を空にして終わる

### SRS レビュー（毎日、固定枠）

1. dashboard で due ノートを確認
2. カードを recall → 自己評価
3. 失敗したカードは間隔をリセット
4. レビュー時間が長くなりすぎたら新規追加を絞る

### 合成（週次）

1. 直近1週間で書いたノートを横断
2. **共通テーマ・矛盾・空白**を探す
3. 必要なら統合ノート（MOC: Map of Content）を作成
4. ここで [P08 認知ワーク](./p08-deep-work.md) のテーマが進化することがある

### 庭園整備（月次）

1. 30日以上 review されていないノートを抽出
2. 統合候補・廃棄候補・継続候補に分類
3. リンク切れ・命名揺れを修正
4. dashboard の retention 平均を確認

## 命名・構造のルール

- ファイル名は **概念名そのもの**（日付ベース禁止）
- frontmatter に `title` / `tags` / `created_at` / `updated_at` を必ず置く
- 比較・対立は edge にしない（独立ノート同士をリンクで関係付ける）
- テーブル内で wikilink alias パイプを使わない（graph 構造を事実関係の地図に保つ）

## 違反検知

- **inbox 滞留**: 投入から48時間以上残っているメモがある
- **SRS 滞留**: due が3日以上溜まっている
- **孤立ノート**: 他ノートへのリンクが0のノートが増えている
- **検索失敗**: 「あれどこに書いたっけ」が週2回以上発生
- **重複**: 同じ概念のノートが2つ以上ある
- **命名揺れ**: 同概念で複数の表記が並走している

## 復旧手順

1. inbox 滞留 → 加工枠を朝の固定時間に移動（[P08](./p08-deep-work.md) 開始前）
2. SRS 滞留 → 1日のレビュー件数を半減してでも毎日継続
3. 孤立ノート → 週次合成でリンクを補強、改善しなければ廃棄
4. 検索失敗 → タグ運用 or 命名規則を見直す
5. 重複 → 月次の庭園整備で統合
6. 全層崩壊 → 新規追加を1週間止め、既存ノートの整理だけに専念

## 現行実装（参考）

- vault: `O6lvl4-knowledge/vault/`（Markdown + frontmatter）
- インデックス自動生成: graph-garden
- SRS: awen（dashboard で retention 可視化）
- 編集: 任意エディタ

ツールが変わってもプロトコル自体は不変。「キャプチャ → 加工 → 定着 → 再利用」の4層を維持できる手段ならよい。
