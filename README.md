# O6lvl4 Protocol

知的労働を最大化するための、肉体・睡眠・食事・認知の運用プロトコル集。

各プロトコルは **目的 / 不変条件 / パラメータ / 実行手順 / 違反検知 / 復旧手順** の固定構造で記述する。本体は [protocols/](./protocols/) 以下に種類ごとに分割。

## 設計原則

- **主役は脳、肉体はインフラ**。トレーニングは「翌日の集中力が上がる強度」を上限とする
- **可逆性より定常性**。極端な増量・減量サイクルを禁止し、年間を通じて同じ体組成を維持する
- **回復を変数ではなく定数にする**。睡眠・食事タイミングは固定。運動量を可変にする
- **計測なきものは運用しない**。ただし指標は最小限に絞る

## 目指す体型

一言でいうと、**「軽くて、血流がよくて、筋肉は十分あり、脂肪は乗りすぎていない体」**。

筋肉隆々の戦車ではなく、長時間高性能に動く軽量ワークステーション。マラソン選手ほど細くなく、格闘家ほどゴツくなく、クライマー・中距離走者・サッカー選手・体操寄りのしなやかな体。

筋肉は「作品」ではなく「発電所・冷却装置・姿勢保持装置」として扱う。

### なぜ筋肉隆々が知的労働に向かないか

筋肉量そのものではなく、**それを維持・拡大するための生活コスト**が脳と衝突する。

- 高ボリューム筋トレで中枢疲労が残る
- 食事量が多く、食後に眠くなりやすい
- 増量・減量でメンタルや集中力が揺れる
- 体が重くなり、日中の軽快さが落ちる
- トレーニングと回復に認知リソースを食われる

## ターゲット体組成

| 指標 | 目標レンジ | 判定頻度 |
|---|---|---|
| 体脂肪率 | 13〜17%（男性想定） | 週1回・同時刻 |
| 体重変動 | ±2kg以内 / 月 | 週3回・朝起床後 |
| ウエスト | 身長×0.45以下 | 月1回 |
| 安静時心拍 | 55〜65bpm | 毎朝 |
| HRV | 個人ベースライン±20%以内 | 毎朝 |

**赤信号ルール**: 体脂肪率が3週連続で上限超え、または安静時心拍が10bpm 以上恒常的に上昇 → 強度・食事を即見直し。

## 有酸素は毎日やるべきか

**結論：低強度（Zone 2 以下）は毎日やってよい。中〜高強度は週2回上限。**

- Zone 2（心拍最大の60〜70%）はミトコンドリア新生・脳血流・気分安定に効き、回復を妨げない
- 「毎日同じ強度」は危険。**強度を分散させる**のが正解
- 完全休養は週0〜1日でよいが、その日も10〜30分の散歩は推奨

つまり「毎日**動く**」は正解、「毎日**走る/追い込む**」は間違い。Zone 2 の具体例と詳細配分は [P02 有酸素プロトコル](./protocols/p02-cardio.md) を参照。

## 唯一の最適化対象

成果指標は1つだけに絞る：

> **先週、深い集中ができた合計時間**

これが減っていれば、運動強度・食事・睡眠のいずれかが過剰または不足している。鏡や数字（体重・体脂肪率）ではなく、この1指標に収束させる。

## 12週間の立ち上げ計画

- **週1〜4**: 睡眠固定（P01）+ 毎日 Zone 2 30分（P02）+ 筋トレ週2回・軽め（P03）
- **週5〜8**: 食事構造（P04）を固定、ディープワーク帯（P08）に運動を組み込まない設計に変更
- **週9〜12**: 計測指標（P10）で微調整、強度を「集中時間」が伸びる範囲で最適化

12週後に体組成・集中時間・睡眠スコアを評価し、以後は維持運用に移行。

## 設計の核

> **強度を分散し、リズムを固定し、観測は最小化する。**

各プロトコルは独立に違反検知でき、復旧手順は一本道。迷いを設計から排除する。
